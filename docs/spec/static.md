# ObjectiveR `static` 关键字规范

## 1. 关键语义

- `static` 用于将被标识值提升为静态可访问值。
- 被 `static` 标识且可见性为 `public`（显式或默认）时，该值在其语言空间内可访问。
- 若被 `private static` 标识，则该值仅在当前 `.obr` 文件内全局可访问，不得跨文件访问。

## 2. 与可见性的关系

- 未显式标注可见性时，`static` 目标默认可见性为 `public`。
- `public static`：在语言空间访问规则允许的前提下，可跨文件访问。
- `private static`：始终仅限当前文件访问，即使语言空间开放也不可跨文件访问。

## 3. 动态逃逸语义

- `static` 允许在函数体内对值进行静态标识。
- 在函数体内使用 `private static` 时，允许创建“文件内可持续访问”的动态静态值。
- 使用 `public static` 时，动态静态值可在语言空间内被访问（受 `#LINK` 与可见性共同约束）。

## 4. 不可逆规则

- 对同一值的 `static` 标识是不可逆的。
- 同一值一旦被标识为 `static`，不得再次对该值执行 `static` 标识。
- 解释器在执行静态标识前必须进行重复标识检查。
- 发现重复标识时必须报错并拒绝该次标识行为。

## 5. 唯一性规则

- 静态标识值在“可互访范围”内必须保持唯一。
- “可互访范围”必须按 `#LINK` 访问判定定义：仅当文件间访问在 `#LINK` 规则下成立时，才视为处于同一可互访范围。
- 若两个文件 `A` 与 `B` 彼此可访问，且 `A` 已将值 `C` 标识为 `static`，则 `B` 不得再次将 `C` 标识为 `static`。
- 若文件 `D` 与 `A` 不在同一可互访范围（不关联），则 `D` 可独立将其同名值 `C` 标识为 `static`。

### 5.1 判定顺序

- 先依据 `#LINK` 判定文件是否处于可互访范围。
- 再在该可互访范围内执行静态标识唯一性检查。
- 若不在同一可互访范围，不构成 `static` 唯一性冲突。

## 6. 报错条件

以下情况必须报错：

- 对同一值重复执行 `static` 标识；
- 在同一可互访范围内出现 `static` 标识值唯一性冲突；
- 其他违反本规范第 4 节与第 5 节的行为。

## 7. 静态表（Static Table）

- 解释器必须维护静态表，用于登记所有被 `static` 标识的值。
- 任一值一旦被 `static` 标识，必须写入静态表。
- 静态表记录项必须包含该值的作用空间信息，用于后续访问判定与冲突检查。

### 7.1 实例唯一性

- 静态表在全局语义上有且只有一个。
- 等价约束：每个解释器实例必须且只能持有一个静态表。
- 同一解释器实例内，不得存在多个并行静态表。
