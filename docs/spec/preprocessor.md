# ObjectiveR 预处理指令规范

## 总则

- Obr 支持预处理指令。
- 预处理指令必须位于 `.obr` 文件开头区域。
- 以 `#` 开头的行均视为预处理指令。
- 预处理指令后必须紧跟可识别的处理指令。
- 预处理指令不强制固定排版格式，但凡无法被解析，必须报错。
- 由于不同 Obr 语言版本可用的预处理指令集合是确定的，遇到未支持指令应报错。
- 预处理阶段失败时，必须终止当前文件加载。

## 通用解析规则

- 指令名与参数之间至少一个空白字符分隔。
- 允许多个空白字符。
- 指令名大小写敏感。
- 指令名必须完整且连续，不可拆分。

## `#VERSION` 指令

用于标识当前 `.obr` 文件的语言版本。

- 非强制指令：文件可以不写 `#VERSION`。
- 若未标识版本，默认采用当前解释器可解释执行的 Obr 语言版本。
- 若显式标识版本，解释器必须验证当前是否兼容或已适配该版本；不兼容则报错。

正确示例：

```text
#VERSION 1
# VERSION 1
#   VERSION   1
```

错误示例（无法解析）：

```text
#VERS ION 1
#VERSION1
```

说明：

- `#VERS ION 1` 中指令名被拆分，导致无法识别为 `VERSION`。
- `#VERSION1` 中指令名与参数粘连，导致无法分离出版本值。

## `#LINK` 指令（语言空间）

用于定义“哪些 `.obr` 文件可以访问当前文件（current file）中的公开符号”。

### 1) 核心定义（无歧义）

- **当前文件**：正在解析 `#LINK` 的文件，记为 `F`。
- **访问方文件**：尝试调用 `F` 中符号的文件，记为 `C`。
- **`#LINK` 列表**：`F` 在预处理阶段解析得到的一组路径项，记为 `L`。
- **判定目标**：仅判定 `C -> F` 是否允许访问。  

访问判定规则（唯一规则）：

- 若 `L` 为空（`F` 未声明 `#LINK`），则 `F` 的默认空间为 `/`（项目根空间）。
- 若 `L` 非空，则 `C` 只要命中 `L` 中任一项即允许访问 `F`，否则拒绝访问。
- 命中逻辑为“并集”语义（OR），不存在优先级覆盖。

### 2) 路径项类型与命中规则

`L` 中每个路径项只能是以下两类之一：

- **空间路径项（目录空间）**：如 `/`、`/lib`、`/lib/`
  - 规范化后 `/lib` 与 `/lib/` 等价。
  - 命中条件：`C` 位于该目录空间下（包含其子目录中的 `.obr` 文件）。
- **文件路径项（文件级）**：如 `/main.obr`、`/lib/util.obr`
  - 命中条件：`C` 的规范化绝对路径与该项完全相等。

### 3) 默认行为

- `F` 未声明 `#LINK` 时，等价于 `#LINK /`。
- 因此，默认允许项目内任意 `.obr` 文件访问 `F`。

### 4) 解析与格式规则

- 指令格式：`#LINK <item>(,<item>)*`
- 允许逗号后换行续写；续写行在语义上与上一行拼接为同一 `#LINK` 参数串。
- 逗号续写判定必须满足：逗号后不存在除空格字符外的其他字符；若存在其他字符，则当前行在本行结束。
- 多个 `#LINK` 指令同时出现时，解释器必须将其参数项合并为同一并集列表 `L`。
- 任何空项或无法归类为“空间路径项/文件路径项”的项都必须报错。
- 同一路径项重复出现时，按同一项处理，不得改变访问结果。

报错示例（必须失败）：

```text
#LINK
#LINK ,
#LINK /lib,,/main.obr
#LINK lib
```

### 5) 规范化规则（必须一致）

- 所有 `#LINK` 路径必须视为“以项目根为起点”的绝对逻辑路径。
- 分隔符统一为 `/`。
- 去除冗余尾部 `/`（根 `/` 除外）。
- 比较前需完成同一规范化流程，否则实现不一致。

### 6) 你给出的示例（规范化解释）

示例输入：

```text
#LINK /
#LINK /lib
#LINK /main.obr
#LINK /main.obr,/lib/util.obr
#LINK /lib/util.obr,
      /lib/math.obr
```

解释：

- `#LINK /`：当前文件对项目内所有 `.obr` 开放访问。
- `#LINK /lib`：仅 `/lib` 空间及其子空间中的文件可访问当前文件。
- `#LINK /main.obr`：仅 `main.obr` 可访问当前文件。
- `#LINK /main.obr,/lib/util.obr`：`main.obr` 与 `lib/util.obr` 均可访问（并集）。
- 逗号换行写法与单行等价。

### 7) 典型判定示例

目录：

```text
/main.obr
/lib/util.obr
/lib/math.obr
```

- 若 `F=/lib/util.obr` 且 `#LINK /lib`，则：
  - `C=/main.obr` 不可访问 `F`；
  - `C=/lib/math.obr` 可访问 `F`。
- 若 `F=/lib/math.obr` 且 `#LINK /main.obr`，则仅 `main.obr` 可访问 `F`。
- 若 `F=/main.obr` 且无 `#LINK`，按默认 `#LINK /` 处理，项目内文件均可访问 `F`。
