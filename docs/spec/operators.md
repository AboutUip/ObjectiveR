# ObjectiveR 运算符与隐性转化规范

本文件用于固化 ObjectiveR 与 JavaScript 的“例外/偏离语义”，并对若干运算符给出无歧义的判定规则。除本文件明确列出的例外/偏离语义外，其余表达式运算符的求值语义与 JavaScript 对齐，且优先级/结合性与 JavaScript 对齐。

## 0. 运算符集合（与 JavaScript 对齐）

ObjectiveR 的表达式运算符符号集合与 JavaScript 对齐。下列运算符在本文件中给出无歧义的判定规则：

- 自增自减：`++`、`--`
- 一元：`!`、`+`、`-`、`~`
- 幂：`**`
- 算术：`+`、`-`、`*`、`/`、`%`
- 位运算：`&`、`|`、`^`、`<<`、`>>`、`>>>`（以及 `~`）
- 关系：`<`、`<=`、`>`、`>=`
- 相等：`==`、`!=`
- 逻辑：`&&`、`||`

- 例外/偏离语义（必须以本文件为准）：

- 相等比较：`==/!=` 采用严格相等语义（不做宽松比较，且类型不一致必须报错）。
- `+`：不允许字符串拼接（任一侧为字符串时必须报错）。
- 逻辑运算：`&&/||` 的结果强制为 `boolean`。

## 1. 隐性转化（Implicit Coercion）

- 运算符在遇到左右操作数类型不一致时，隐性转化采用 **JS 风格**。
- 布尔上下文/逻辑相关的布尔判定使用 **JS truthy/falsy** 规则。

说明（范围）：

- “隐性转化”适用于需要进行类型适配的运算场景。
- 在本规范的相等比较部分，已单独选择了“严格比较”，因此相等比较不会因为隐性转化而变成宽松语义（见第 2 节）。

## 2. 相等与不等运算（`==`/`!=`）

- 所有相等相关运算均采用严格语义：`==/!=`。
- 即：不执行类型转换；当两侧类型不一致时，该比较必须报错（或按你后续指定的统一错误策略处理）。
- 当两侧类型一致时，比较按类型对应的值语义执行。

## 3. 加法运算符 `+`

- 当 `+` 的两侧操作数类型不一致时：
  - 若两侧均为数值类型，则按数值相加执行；
  - 若任一侧为字符串类型，则必须报错，不允许字符串拼接。

## 4. 逻辑运算 `&&/||`（结果类型）

- `&&/||` 的结果必须为 `boolean`。
- 在布尔上下文中，非 `boolean` 类型会按 JS truthy/falsy 规则转换为布尔值参与运算。

## 5. 自增自减运算符 `++/--`

### 5.1 可操作对象（L-Value）

- `++` 与 `--` 只能作用于可赋值目标（L-Value），不可作用于字面量或不可写表达式结果。
- 若目标对象因可见性/访问规则不可访问，则必须报错（在进入运算前失败）。

### 5.2 前缀/后缀形式

- 前缀：`++x` / `--x`  
  - 先更新目标值，再把更新后的值作为表达式结果。
- 后缀：`x++` / `x--`  
  - 先把更新前的值作为表达式结果，再更新目标值。

### 5.3 类型约束

- `++/--` 只能作用于数值类型：`byte/short/int/long/float/double`。
- 其它类型（`boolean/string/char/复杂类型/null/undefined`）一律必须报错（类型错误）。

### 5.4 运算规则

- `++/--` 等价于对目标执行 `+1` 或 `-1` 的数值运算并写回。
- 对 `byte`：允许集合为 `{0,1}`，执行 `+1/-1` 后必须落回 `{0,1}`（按取模 2 循环定义）。

### 5.5 返回值类型

- `++/--` 的表达式结果类型与目标类型相同。

## 6. 一元与算术运算（`! + - ~ ** * / %`）

### 6.1 一元 `!`

- `!` 的结果为 `boolean`。
- 操作数按 JS truthy/falsy 规则转换为布尔值后取反。

### 6.2 一元 `+` / `-`

- 一元 `+/-` 的操作数参与 JS 风格隐性数值适配后计算。
- 结果类型按后续“数值运算与提升”专题规则确定；本文件不展开。

### 6.3 `~`

- `~` 结果类型为 `int`。
- `~` 对操作数按 JS 风格适配为 32-bit 位运算整数后计算。

### 6.4 幂 `**`

- `**` 的两侧操作数按 JS 风格隐性数值适配后计算。
- 结果类型按后续“数值运算与提升”专题规则确定；本文件不展开。

### 6.5 乘除取模 `* / %`

- `* / %` 两侧操作数按 JS 风格隐性数值适配后计算。
- 发生未定义行为（如除零）由后续专题补充；本文件不展开除零等未定义行为。

## 7. 位运算与移位（`& | ^ << >> >>>`）

- 位运算与移位两侧操作数按 JS 风格适配为 32-bit 位运算整数后计算。
- 结果类型为 `int`。

## 8. 关系运算（`< <= > >=`）

- 关系运算结果为 `boolean`。
- 两侧操作数按 JS 风格隐性适配后比较。
- 当比较需要字符串字典序时，按字符串规则比较；否则按数值比较（与 JavaScript 对齐）。

## 9. 优先级与结合性（与 JavaScript 对齐）

为消除解析歧义，解析必须采用以下优先级（从高到低），且结合性按条目说明：

1. 后缀 `x++` / `x--`
2. 前缀 `++x` / `--x`，以及一元 `! + - ~`
3. 幂 `**`（右结合）
4. 乘除取模 `* / %`
5. 加减 `+ -`
6. 移位 `<< >> >>>`
7. 关系 `< <= > >=`
8. 相等 `== !=`
9. 位与 `&`
10. 位异或 `^`
11. 位或 `|`
12. 逻辑与 `&&`
13. 逻辑或 `||`

- 未在本表中列出的其它 JavaScript 表达式运算符，其优先级/结合性按 JavaScript 对应规则确定。
