# ObjectiveR 作用域与标识符解析规范

## 0. 标识符定义（词法层）

ObjectiveR 的标识符用于命名变量、函数、类型（如存在）、以及其他可命名实体。

### 0.1 字符与形式

- 标识符大小写敏感。
- 标识符必须匹配如下结构（ASCII 规则）：
  - 首字符：英文字母（`A-Z` 或 `a-z`）或下划线（`_`）
  - 后续字符：英文字母、数字（`0-9`）或下划线（`_`）
- 标识符不得以数字开头。
- 标识符中不得包含空白字符或除字母/数字/下划线以外的字符（例如 `-`、`.`）。

### 0.2 关键字与保留字

以下词在词法上不得作为普通标识符使用（它们视为关键字/内建常量/语句关键字）：

- 访问修饰关键字：`public`、`private`
- 关键字：`static`
- 预处理相关：`#VERSION`、`#LINK`（以 `#` 开头的指令不属于标识符体系）
- 内建常量/特殊值：`true`、`false`、`null`、`undefined`
- 基础类型关键字：`byte`、`short`、`int`、`long`、`float`、`double`、`char`、`string`、`boolean`
- 特殊类型/返回类型关键字：`void`（无值类型，仅用于允许的声明位，不得作为普通标识符）
- moduleR 函数头声明关键字：`deRfun`
- moduleR 函数头特性前缀：`@Overwrite`、`@Callfun`
- moduleR 导入关键字：`import`
- moduleR 命名空间关键字：`namespace`（**仅**允许出现在 `.mr` 中用于声明命名空间；不得作为普通标识符）

## 1. 作用域层级

ObjectiveR 至少存在以下作用域层级（按从内到外的查找顺序）：

1. **语句块作用域（Block Scope）**：由成组语句构成的块（例如函数体内部的嵌套代码块）。
2. **函数作用域（Function Scope）**：函数形参与函数体内声明的局部标识符。
3. **文件作用域（File Scope）**：`.obr` 文件内的文件级声明（包括函数、静态值等）。
4. **语言空间作用域（Language Space Scope）**：由 `#LINK`/作用空间规则决定，在同一语言空间内可互相访问的公共符号集合。

本规范不讨论语法形态细节；仅规定标识符可见性与解析顺序。

## 2. 标识符解析顺序

当程序需要解析一个标识符 `id` 时，解析按以下顺序进行，直到命中为止：

1. 在当前最内层语句块作用域查找。
2. 若未命中，在当前函数作用域查找。
3. 若未命中，在当前 `.obr` 文件作用域查找。
4. 若未命中，才在语言空间作用域内查找“可访问且可见”的符号。

若以上均未命中，必须报错（标识符未定义）。

### 2.1 限定名 `::`（命名空间路径）

- 当语法位置要求**限定名**（形如 `a::b::c`，见 [`moduleR.md` 第 6.3 节](moduleR.md)）时，解释器**不得**按本节第 1～4 步将 `a` 当作普通标识符在局部/文件作用域中解析为变量等；必须在**当前 `.obr` 已加载/解析的 `.mr` 集合**（含隐式 `system.mr`）所导出的命名空间与符号表中，按 `::` 分段解析。
- 限定名的词法形式必须为 **`标识符(::标识符)+`**；**不得**出现前导 `::`、尾随 `::` 或连续 `::`（见 [`moduleR.md` 第 6.3.3 节](moduleR.md)）。
- 若限定名无法解析到唯一目标（未声明、歧义或与可见性/`#LINK` 冲突），必须报错。

## 3. 遮蔽（Shadowing）

- 若在更内层作用域中声明了与外层同名标识符，则内层声明遮蔽外层同名标识符。
- 解析始终遵循“最内层优先”，因此遮蔽行为不会导致外层符号被隐式修改。

## 4. 函数形参作用域

- 函数形参的作用域覆盖整个函数体。
- 形参在同一函数作用域内遵循“遮蔽”规则。

## 5. 局部变量生命周期

- 局部变量在其声明位置开始可见，并在离开其所属语句块作用域时失效。
- 局部变量失效后，不允许在后续代码中被引用（引用必须在解析阶段被视为未定义）。

## 6. 默认初始化（与 `undefined/null` 的关系）

若局部变量被创建但未显式赋值：

- 若其声明类型属于“基础类型”（见 `types.md` 的基础类型集合），则默认值为 `undefined`。
- 若其声明类型属于“复杂类型”（见 `types.md` 的基础/复杂归类），则默认值为 `null`。

## 7. 与 `#LINK` 与可见性关键字的交互

- 文件/语言空间内的跨文件访问必须满足：
  1. 访问方文件到目标文件的 `#LINK` 命中；
  2. 目标符号满足 `public/private` 的可见性限制；
  3. 目标静态/非静态语义满足 `static` 规则。
- 本规范只规定解析顺序；跨文件是否允许访问由 `moduleR`/`preprocessor`/`visibility`/`static` 等专题规范共同判定。

## 7.1 `static`（存储类别）与 `public` / `private`（可见性）

- **`static` 表示静态存储类别**；**`public` / `private` 表示访问可见性**。二者正交：`public static` 先由 `static` 得到静态实体，再由 `public` 标识为公开（详见 [`visibility.md` 第 8 节](visibility.md)）。
- 函数内 `static var`：槽位跨调用持久，但若**再次执行**到同名 `static var` 声明则须报错，见 [执行模型](runtime.md) §5.1（与 C/Java static 局部「只初始化一次」不同）。

## 8. 报错条件（必须）

当发生以下情况时，必须报错并终止语义处理：

- 解析到标识符但未在任何作用域层级命中（标识符未定义）。
- 在同一作用域层级内出现重复声明且无法确定唯一含义（重复声明）。
