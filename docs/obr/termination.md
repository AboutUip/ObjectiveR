# ObjectiveR 语句终止与换行规范

## 1. 通用终止规则

- 在一般语法场景下，ObjectiveR 使用分号 `;` 作为表达式或语句结束标识。
- 未按规则出现结束标识的语句必须视为语法错误。

## 2. 特殊终止规则

- 特殊语法结构可定义自身结束规则，不强制使用分号 `;`。
- 预处理指令属于特殊结构，按“行结束”判定，不按分号结束。
- `import` 语句属于一般语法结构，仍按分号 `;` 结束。
- **`namespace` 块**（仅 `.mr`）：以闭合 `}` 结束；`}` 后**不得**出现分号 `;`（若出现，必须视为语法错误）。见 [`moduleR.md` 第 6.3.2 节](moduleR.md)。

## 3. 预处理指令的行结束判定

设当前预处理行在参数区包含逗号 `,`，则必须按以下规则判定该行是否结束：

- 若逗号后存在除空格字符外的其他字符，则当前行在本行结束。
- 若逗号后不存在除空格字符外的其他字符，则当前行不结束，必须由下一行继续作为同一条预处理参数串。

## 4. 换行续写规则

- 仅当满足“逗号后无非空格字符”时，允许将下一行视为当前预处理行的续写内容。
- 换行续写后的语义必须与单行连续书写语义完全一致。
- 不满足续写条件时，下一行必须按新行独立解析。
