# ObjectiveR 运算符与隐性转化规范

本文件用于固化 ObjectiveR 与 JavaScript 的“例外/偏离语义”，并对若干运算符给出无歧义的判定规则。除本文件明确列出的例外/偏离语义外，其余表达式运算符的求值语义与 JavaScript 对齐，且优先级/结合性与 JavaScript 对齐。

## 0. 运算符集合（与 JavaScript 对齐）

ObjectiveR 的表达式运算符符号集合与 JavaScript 对齐。下列运算符在本文件中给出无歧义的判定规则：

- 自增自减：`++`、`--`
- 一元：`!`、`+`、`-`、`~`
- 幂：`**`
- 算术：`+`、`-`、`*`、`/`、`%`
- 位运算：`&`、`|`、`^`、`<<`、`>>`、`>>>`（以及 `~`）
- 关系：`<`、`<=`、`>`、`>=`
- 相等：`==`、`!=`
- 逻辑：`&&`、`||`
- 条件：`?:`（三元；语法与语义见 [`control-flow.md`](control-flow.md) §3）

- **命名空间限定 `::`**：用于**限定名**（如 `test::foo()`），见 [`moduleR.md` 第 6.3 节](moduleR.md) 与 [`scope.md` 第 2.1 节](scope.md)；**不属于** JavaScript 表达式运算符集合，亦**不参与**本节下文所列优先级表。

- 例外/偏离语义（必须以本文件为准）：

- 相等比较：`==/!=` 采用严格相等语义（不做宽松比较；**禁止**为比较而做隐性类型转换；允许的类型组合见第 2 节）。
- **二元 `+`**：**不**按 §1 中「任意运算都先套 JS 风格隐性适配」的笼统表述处理；**仅**按第 3 节（字符串拼接 / 纯数值加法）判定。
- `+`：**至少一侧**为 `string` 或 `char` 时，为 **字符串拼接**（`char` 先转为 `string`）；详见第 3 节。
- 逻辑运算：`&&/||` 的结果强制为 `boolean`。

## 1. 隐性转化（Implicit Coercion）

- 运算符在遇到左右操作数类型不一致时，**在未被下列条文排除的前提下**，隐性转化采用 **JS 风格**。
- **适用范围**：上述「JS 风格」**仅**用于 **§1.2**、**§3.2**（纯数值路径）、**§6.4–§6.5**、**§7** 中已写明之处；**不**适用于 **§3.1**（拼接判定）、**§2**、**§8**。

- **排除**：**二元 `+`** 仅按 §3 判定（字符串拼接 vs 数值加法），**不**将 §1 的笼统 JS 适配套用在二元 `+` 上。
- **排除**：**关系** `<` `<=` `>` `>=` 与 **相等** `==` `!=` 禁止为比较而做隐性类型转换；见第 2 节、第 8 节。
- 布尔上下文/逻辑相关的布尔判定使用 **JS truthy/falsy** 规则，**但** `char` 与 `byte` 按 §1.1 的 **`ToBoolean`** 表执行（与 JS 无原生 `char` 时的差异已固定）。

说明（范围）：

- “隐性转化”适用于需要进行类型适配的运算场景（如若干一元/算术/位运算；见 §6、§7 等）。
- 在本规范的相等比较部分，已单独选择了“严格比较”，因此相等比较不会因为隐性转化而变成宽松语义（见第 2 节）。
- **`if` / `?:` 的条件**：须先对条件值做 `ToBoolean`，语义见 [`control-flow.md`](control-flow.md) §1.2；`char` 细则见同文件 §4。

### 1.1 各类型在布尔上下文中的真值（`ToBoolean`）

下列规则用于：`!` 的操作数、`&&` / `||` 的短路判定、**`if ( Cond )`** 与 **`?:` 的 `Cond`** 等**布尔上下文**。若某类型标注为「禁止」，则该类型表达式**不得**出现在上述布尔上下文中，必须**静态报错**。

| 类型 | 转为 `true` | 转为 `false` |
|------|---------------|----------------|
| `boolean` | `true` | `false` |
| `byte` | `1` | `0` |
| `short` / `int` / `long` | 任意非零整数值 | **`0`**（含与零相等的整数值） |
| `float` / `double` | 任意非零且非 `NaN` 的值 | **`0.0`、`-0.0`**、`NaN` |
| `char` | 任意非空字符（码点非 0） | **`''`**（空字面量）、**`'\0'`**（与 `''` 同值，码点 0） |
| `string` | 长度 ≥ 1 的非空串 | **`""`**（空字符串） |
| `undefined` | — | **`false`**（无真值分支） |
| `null` | — | **`false`**（无真值分支） |
| 复杂类型引用 | 非 `null` 引用 | **`null` 引用** → **`false`** |
| `void` | **禁止** | **禁止** |

- **`undefined` 与 `null`**：在布尔上下文中经 `ToBoolean` **均为** `false`（与 JavaScript 中二者均为 falsy 一致）。
- **浮点 `NaN`**：一旦出现，`ToBoolean` 为假（产生与运算细节见类型与运算专题）。

- **非布尔上下文的隐性转化**（如二元数值混合）：仍按 **JS 风格**；**相等** `==`/`!=` 不参与 `ToBoolean`（见第 2 节）。

### 1.2 数值运算与提升（与 JavaScript 对齐）

- 在 **§3.2** 纯数值加/减、**§6.4** `**`、**§6.5** `* / %`、**§7** 位运算与移位中，对数值类型的混合与适配，按 **JavaScript** 对 `Number` 的转换与二元运算规则执行（与 `Number` 运算一致）。
- **`byte` 与其它数值类型**：在以上**二元**运算中，若一侧为 **`byte`**、另一侧为 **`short`/`int`/`long`/`float`/`double`**（即**非**两侧同为 `byte`），须**静态报错**。**两侧均为 `byte`** 时**不**套用本节与 JS 的混合规则，见 **§3.2** 专条。
- **不**改写 §2、§8；**二元 `+`** 仍先由 §3 判定为拼接或纯数值，再走 §3.2。
- **与调用加宽**：函数**调用**实参的数值加宽见 [`moduleR.md`](moduleR.md) §7.9 第 2 步；**不**视为本条对表达式运算的放宽。

## 2. 相等与不等运算（`==`/`!=`）

- 所有相等相关运算均采用严格语义：`==/!=`。
- **禁止**为比较而做隐性类型转换：当两侧**静态类型**不一致时，必须报错。
- **两侧静态类型须一致**时，比较按类型对应的值语义执行。
- **允许参与 `==`/`!=` 的类型**（两侧须为**同一**类型名）：`byte`、`short`、`int`、`long`、`float`、`double`、`string`、`char`。
- **`string`/`char` 与数值类型**、`string` 与 `char` **不得**混用（须报错）。
- **`byte`/`short`/`int`/`long`**：按整型值相等。
- **`float`/`double`**：按 **IEEE 754**；`NaN` 与任意值（含 `NaN` 自身）的 `==` 为假、`!=` 为真；`+0` 与 `-0` 的 `==` 为真。
- **`string`**：**引用相等**——仅当两侧为**同一** `string` 实例时为真；内容相同但非同一实例为假。
- **`char`**：按字符值（码点）相等。

## 2.1 与函数调用的关系

- 实参与重载分辨遵循 [`moduleR.md`](moduleR.md) **§7.9**（**先**完全一致，**否则**调用点数值加宽，见该节）；**与** §3 二元 `+` 无关——不得用「`+` 一侧为 `string`」推断重载。

## 3. 加法运算符 `+`

- **与重载调用**：本节对二元 `+` 的判定**不改变**函数调用的重载匹配；见 §2.1 与 [`moduleR.md`](moduleR.md) §7.9。

### 3.1 字符串拼接（至少一侧为 `string` 或 `char`）

- 当**左或右**任一操作数的静态类型为 **`string`** 或 **`char`** 时，`+` 表示 **字符串拼接**，**结果类型**为 **`string`**。
- **`char` → `string`**：将该字符按**单码点 Unicode 文本序列**（长度为 1 的 `string`）表示（与字面量中该字符的书写语义一致；如 `'\0'` 转为含 U+0000 的一字符串）。
- **另一侧为数值**（`byte` / `short` / `int` / `long` / `float` / `double`）：须先按**十进制文本**规则转为 `string`（整数无前导零歧义；**浮点**具体格式**语言规范暂不固定**，由实现定义，后续 Obr 版本可收紧）。
- **两侧均为 `string`**：直接首尾拼接。
- **一侧 `string` 一侧 `char`**：将 `char` 转 `string` 后拼接。
- **两侧均为 `char`**：分别将两侧 `char` 转 `string` 后拼接（结果仍为 `string`）。
- **与 `boolean` / `null` / `undefined` / 复杂类型**混合参与 `+`：必须**静态报错**（类型错误），**不得**静默拼接。

### 3.2 纯数值加法与二元 `-`

- 当**两侧**均为**数值类型**（`byte`…`double`），且**均非**本节的「字符串拼接」情形时：
  - **两侧均为 `byte`**：按算术加/减求值后，**结果类型**为 **`byte`**，值对 **2** 取模落回 `{0,1}`（与 §5.4 `++/--` 循环一致）。
  - **否则**（且非 §1.2 禁止的 `byte` 混用）：提升规则见 §1.2。
- **二元 `-`**（纯数值）：类型与 **`byte`** 规则同 **加法**。

### 3.3 复合赋值（`+=` / `-=` / `*=` / `/=` / `%=`）

- **`+=`**
  - **左侧**为 **`string`**：与 **`lhs = lhs + rhs`** 在 **`+` 为拼接**时一致（`rhs` 按 §3.1；不满足 §3.1 须报错）。
  - **左侧**为 **`char`**：**静态报错**（非法目标）。
  - **左侧**为 **`byte`/`short`/`int`/`long`/`float`/`double`**：数值 `+=`；**`rhs` 与 `lhs + rhs` 须满足 §1.2 与 §3.2**（**`byte` 左侧**时 **`rhs` 须为 `byte`**，否则静态报错）。
- **`-=` `*=` `/=` `%=`**
  - **左侧**须为 **`byte`/`short`/`int`/`long`/`float`/`double`** 且可赋值；语义为 **`lhs = lhs op rhs`**，**`op`** 与对应二元运算符同 §1.2 / §6–§7；**`byte` 左侧**时 **`rhs` 须为 `byte`**（否则静态报错）。
  - **左侧**为 **`string`** 或 **`char`**：**静态报错**（**仅** `+=` 允许 `string` 左侧）。

## 4. 逻辑运算 `&&/||`（结果类型）

- `&&/||` 的结果必须为 `boolean`。
- 短路判定时，对操作数逐次按 §1.1 **`ToBoolean`** 求真假（不再使用未表列的 JS truthy 规则）。

## 5. 自增自减运算符 `++/--`

### 5.1 可操作对象（L-Value）

- `++` 与 `--` 只能作用于可赋值目标（L-Value），不可作用于字面量或不可写表达式结果。
- 若目标对象因可见性/访问规则不可访问，则必须报错（在进入运算前失败）。

### 5.2 前缀/后缀形式

- 前缀：`++x` / `--x`  
  - 先更新目标值，再把更新后的值作为表达式结果。
- 后缀：`x++` / `x--`  
  - 先把更新前的值作为表达式结果，再更新目标值。

### 5.3 类型约束

- `++/--` 只能作用于数值类型：`byte/short/int/long/float/double`。
- 其它类型（`boolean/string/char/复杂类型/null/undefined`）一律必须报错（类型错误）。

### 5.4 运算规则

- `++/--` 等价于对目标执行 `+1` 或 `-1` 的数值运算并写回。
- 对 `byte`：允许集合为 `{0,1}`，执行 `+1/-1` 后必须落回 `{0,1}`（按取模 2 循环定义）。

### 5.5 返回值类型

- `++/--` 的表达式结果类型与目标类型相同。

## 6. 一元与算术运算（`! + - ~ ** * / %`）

### 6.1 一元 `!`

- `!` 的结果为 `boolean`。
- 操作数先按 §1.1 **`ToBoolean`** 转为 `boolean`，再取逻辑非。

### 6.2 一元 `+` / `-`

- 一元 `+` 与一元 `-` 的操作数须为数值类型：`byte` / `short` / `int` / `long` / `float` / `double`。
- **禁止**对 `string` / `char` / `boolean` 等做 JS 式“先转成数字再运算”的隐式转换；非法类型须**静态报错**。
- 合法时，结果类型与操作数类型相同（与 JavaScript 一元 `+`/`-` 在纯数值操作数上一致）。

### 6.3 `~`

- `~` 结果类型为 `int`。
- `~` 对操作数按 JS 风格适配为 32-bit 位运算整数后计算。

### 6.4 幂 `**`

- **两侧均为 `byte`**：将 `0`/`1` 按 **JavaScript `Number`** 求 `**`，再将结果**归约**为 `byte`（对 2 取模至 `{0,1}`）。
- **否则**：两侧操作数按 §1.2 数值适配后计算；结果类型与 **JavaScript `**`** 一致（允许混合数值类型，常见静态推断为 `double`，见 §10 表）。

### 6.5 乘除取模 `* / %`

- **两侧均为 `byte`**：将操作数按 **`Number`** 执行 `*` `/` `%`；**结果类型** `byte`，值对 **2** 取模至 `{0,1}`。除数为 `0` 时仍按下行**整型除零**。
- **否则**：两侧操作数按 §1.2 数值适配后计算。
- **除零**
  - **整型**（`byte`/`short`/`int`/`long`）：除数为 `0` 的 `/` 或 `%` → **运行时错误**（Blink：`E_RT_INTEGER_DIV_ZERO`，见 [`docs/blink/errors.md`](../blink/errors.md)）。
  - **`float`/`double`**：`/` 除数为 `0` 按 **IEEE 754**（`±Infinity`、`NaN` 等）；`%` 除数为 `0` 按 IEEE（常为 `NaN`）。

## 7. 位运算与移位（`& | ^ << >> >>>`）

- **类型**：**恰有一侧**为 **`byte`**、另一侧为其它数值类型时，须**静态报错**。**两侧均为 `byte`**，或**两侧均为非 `byte` 数值类型**且满足 §1.2 时，两侧操作数按 JS 风格适配为 32-bit 位运算整数后计算。
- 结果类型为 `int`。
- **移位量**：右操作数按 **mod 32** 取有效位移（与 JavaScript `<<` / `>>` / `>>>` 一致）。

## 8. 关系运算（`< <= > >=`）

- 关系运算结果为 `boolean`。
- **禁止**为比较而做隐性类型转换。
- **两侧静态类型须一致**，且类型必须为 **`byte` / `short` / `int` / `long`** 之一。
- **不允许** `float` / `double` / `string` / `char` 等参与关系运算（须静态报错）。
- 合法时按该整型的值序比较（与 JavaScript 在**同类型整数**比较上的行为对齐）。

## 9. 优先级与结合性（与 JavaScript 对齐）

为消除解析歧义，解析必须采用以下优先级（从高到低），且结合性按条目说明：

1. 后缀 `x++` / `x--`
2. 前缀 `++x` / `--x`，以及一元 `! + - ~`
3. 幂 `**`（右结合）
4. 乘除取模 `* / %`
5. 加减 `+ -`
6. 移位 `<< >> >>>`
7. 关系 `< <= > >=`
8. 相等 `== !=`
9. 位与 `&`
10. 位异或 `^`
11. 位或 `|`
12. 逻辑与 `&&`
13. 逻辑或 `||`
14. 条件 `?:`（右结合；见 [`control-flow.md`](control-flow.md) §3）

- 未在本表中列出的其它 JavaScript 表达式运算符，其优先级/结合性按 JavaScript 对应规则确定。

**说明**：`?:` 结果类型见 [`control-flow.md`](control-flow.md) §3.2；与二元 `+`、关系/相等（§2、§8）条文独立。

## 10. 与全集差异（实现对照）

相对上文「与 JavaScript 对齐」全集的已实现子集与已知差异；细节见 [`docs/blink/implementation-scope.md`](../blink/implementation-scope.md) 与源码。

| 类别 | 实现子集（BlinkEngine，与 [implementation-scope.md](../blink/implementation-scope.md) 一致） |
|------|----------|
| 赋值与复合赋值 | 支持 `=`、`+=`、`-=`、`*=`、`/=`、`%=`（左侧为单标识符语句）；见 §3.3（`string` 仅 `+=`；`char` 目标复合赋值非法） |
| 自增自减 | **语句**：`++x;`、`x++;`、`--x;`、`x--;`；**表达式**：前缀 `++x`/`--x`、后缀 `x++`/`x--`（操作数为变量；后缀表达式值为旧值，前缀为新值） |
| 算术与一元 | 支持 `+ - * / %`、一元 `+ - ! ~`、幂 `**`（右结合）；**`~`**：操作数为**任意数值 primitive**（含 `byte`/`short`/`int`/`long`/`float`/`double`），结果 **`int`**（`ToInt32`，见 §6.3）；Blink 在静态语义上限制 **`byte` 不得与除 `byte` 外的其它数值混用于二元算术与 `**`**（与 §1.2 全文「JS 风格混用」相比为**更窄**的可运行子集，见 implementation-scope） |
| `if` / `else`、`?:` | 已实现；见 [control-flow.md](control-flow.md) 与 [implementation-scope.md](../blink/implementation-scope.md) |
| 相等 `==` `!=`、关系 `<` `<=` `>` `>=`、逻辑 `&&` `\|\|` | **已**在 `Parser` / `SemanticBinder` / `RuntimeExecutor` 中实现（短路、`?:` 与优先级见 implementation-scope） |
| 位移 `<<` `>>` `>>>` 与二元位运算 `&` `\|` `^` | **已**实现（`Lexer` / `Parser` / `SemanticBinder` / `RuntimeExecutor`；移位量 mod 32、整型按 ECMAScript `ToInt32`/`ToUint32`）；一元按位取反 `~` **已**实现 |
| `+` 字符串拼接 | 已实现（含与数值等允许侧的拼接规则）；见 [implementation-scope.md](../blink/implementation-scope.md) 与 `SemanticBinder#inferAddType` |

完整规范条文仍以本节前文为准；**规范目标**与 Blink 当前子集不一致时，以 [implementation-scope.md](../blink/implementation-scope.md) 为 Blink 的可观察行为说明。
