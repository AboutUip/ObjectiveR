# ObjectiveR 可见性与访问修饰规范

## 1. 关键字

ObjectiveR 定义以下可见性关键字：

- `public`
- `private`

以上关键字大小写敏感。

## 2. 作用位置

- `public` 与 `private` 可附加在部分表达式或声明语句之前。
- 具体可附加的语法对象清单在后续语法规范中补充。
- 在清单补充前，规范仅定义其访问语义与默认行为。

## 3. 默认规则

- 当目标语法对象未显式标注可见性关键字时，默认可见性为 `public`。

## 4. `public` 语义

- `public` 用于显式声明目标对象可被访问。
- 在语言空间访问规则允许的前提下，其他 `.obr` 文件可访问 `public` 目标对象。

## 5. `private` 语义

- `private` 用于声明目标对象仅允许在当前 `.obr` 文件内访问。
- 即使当前文件处于开放语言空间内，`private` 目标对象仍不得被其他 `.obr` 文件访问。
- 即使目标对象具有静态属性（static），`private` 限制仍然生效，不得跨文件访问。

## 6. 优先级与判定关系

- 可见性判定与语言空间判定必须同时满足才允许访问。
- `private` 的文件内限制优先于开放语言空间。
- 因此，语言空间开放并不意味着可绕过 `private`。

## 7. 命名空间内的声明

- 位于 `.mr` 中 `namespace` 体内的 `deRfun` 等声明，在**默认可见性**、`public`/`private` 语义以及 **`#LINK` / 语言空间**约束上，与**未**位于 `namespace` 内的文件级 `deRfun` 声明**相同**；仅符号以**限定名**（形如 `N::foo`）参与解析与访问判定（见 [`moduleR.md` 第 6.3.5 节](moduleR.md)）。

## 8. `static` 与 `public` / `private` 分离

- **`static` 与访问修饰符不是同一类概念。** `public`、`private` 只回答「**谁可见**」；`static` 回答「**存储类别 / 是否静态存储**」（在规范中视为对后续声明施加约束的**运算符级**语义，与「命名用的标识符」区分开）。
- **`public static` / `private static` 的复合语义按两层理解（顺序固定）：**
  1. **`static`**：先作用于随后的变量声明，得到**静态存储**实体（函数内则为本函数静态区槽位；文件级则为文件静态登记等，见 [`scope.md`](scope.md) 与实现专题）。
  2. **`public` 或 `private`**：再对该静态实体施加**可见性**（是否允许跨 `.obr` 等），与步骤 1 正交。
- 单独写 `static var`（无 `public`/`private`）时，按语言默认规则补全可见性（见第 3 节）；**静态性**仍仅由 `static` 决定。

因此：**不得**把 `static` 与 `public` 混成同一种「修饰符」；表述上应区分「静态存储（`static`）」与「公开/私有（`public`/`private`）」。
