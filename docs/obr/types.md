# ObjectiveR 基础数据类型规范

## 1. 支持的基础数据类型

ObjectiveR 支持以下基础数据类型（均为关键字/内建类型）：

- `byte`：只能取 `0` 或 `1`（非 0 即 1）。
- `short`：整型。
- `int`：整型。
- `long`：整型。
- `float`：浮点数。
- `double`：浮点数。
- `char`：单字符类型，以非数字形式表示，使用单引号字面量。
- `string`：字符串类型，使用双引号字面量。
- `boolean`：布尔类型，以关键字 `true` / `false` 表示。

### 1.1 特殊类型 `void`（关键字）

- `void` 为关键字，表示「无值」的特殊类型。
- `void` **不是**可携带值的基础类型，**不存在** `void` 类型的字面量。
- `void` 仅允许出现在需要声明「无返回值」的语法位置（例如函数返回类型）；**不得**用于声明变量、字段或一般表达式的值类型（若出现必须报错，类型错误）。

## 2. 字面量约定

### 2.1 `byte` 字面量

- `byte` 字面量必须为数值 `0` 或 `1`。

### 2.2 `char` 字面量

- `char` 字面量使用单引号 `''` 包裹，且字面量内容必须为“单字符”。
- 例如：`'a'`

### 2.3 `string` 字面量

- `string` 字面量使用双引号 `""` 包裹。
- 空字符串示例：`""`
- 一对双引号之间的字符序列**整体**为该字面量的内容；**引号内的逗号 `,`、空格等不作为**参数分隔或其它语法边界，均按字面计入字符串。例如 **`"Hello,World"`** 表示**一个**字符串（逗号为内容的一部分），而非两个由逗号分隔的记号。

### 2.3.1 函数调用实参类型（诊断阶段）

- 对函数调用：若某实参表达式的类型与对应形参类型在**语义分析阶段**已可确定，且二者不兼容，解释器**必须**在该阶段报错（类型错误），**不得**推迟到运行期才作为唯一手段发现（除非该调用点因语言规则确实无法静态确定类型）。
- 该规则适用于对 `std::rout` 等一切 `deRfun` 调用；数值字面量（如 `555`）与 `[string]` 形参**不兼容**时，须在语义分析阶段报错。

### 2.4 `boolean` 字面量

- `boolean` 字面量为关键字：`true` 或 `false`。

### 2.5 `undefined` 字面量/常量

- `undefined` 为特殊值关键字。
- `undefined` 表示“基础类型值丢失/不存在”（缺失的是值本身，而非引用）。

### 2.6 `null` 字面量/常量

- `null` 为特殊值关键字。
- `null` 表示“复杂类型引用丢失/不存在”（缺失的是引用/指向）。

### 2.7 数值字面量的类型推导

- 若字面量以后缀 `F` 结尾，则该字面量类型为 `float`（`F` 大小写敏感）。
- 若字面量以后缀 `D` 结尾，则该字面量类型为 `double`（`D` 大小写敏感）。
- 默认情况：
  - 含小数部分（即字面量包含小数点 `.`）且未显式写 `F` 或 `D` 时，该字面量类型为 `float`。
  - 不含小数点 `.` 的整数形式且未写 `F` 或 `D` 时，该字面量类型为 `int`。

示例：

```text
1.6F   -> float
1.7D   -> double
1.2    -> float
1      -> int
```

## 3. 位宽与实现约束（按语言规范）

除 `byte` 与 `char` 的特殊约束外，其他数值类型位宽与 Java 平台一致：

- `short`：16 位（与 Java `short` 同尺寸）
- `int`：32 位（与 Java `int` 同尺寸）
- `long`：64 位（与 Java `long` 同尺寸）
- `float`：32 位（与 Java `float` 同尺寸）
- `double`：64 位（与 Java `double` 同尺寸）

- `byte`：1 位（只能取 `0` 或 `1`）。
- `char`：单字符类型，按字符语义实现，非数字实现（不作为数值类型参与数值计算）。

本规范当前不在此章节进一步定义溢出/舍入等运算细节；后续如补充溢出规则，将在同一章节扩展。

## 4. `undefined` 与 `null` 的类型归属与赋值规则

### 4.1 基础类型与复杂类型

- **基础类型（Base Types）**：仅指 `byte/short/int/long/float/double/char/string/boolean` 这组类型。
- **复杂类型（Complex Types）**：指除基础类型以外的所有类型（不含 `void`；`void` 见第 1.1 节，不参与 `undefined`/`null` 的赋值分类）。

### 4.2 赋值限制

- `undefined` 只能赋值给基础类型变量/表达式的目标类型。
- `null` 只能赋值给复杂类型变量/表达式的目标类型。
- 将 `undefined` 赋给复杂类型，或将 `null` 赋给基础类型，必须报错（类型错误，Type Error）。

### 4.3 未初始化变量的默认值

- 若创建了变量，但未提供初始值：
  - 若该变量声明类型为基础类型，则默认值为 `undefined`；
  - 若该变量声明类型为复杂类型，则默认值为 `null`。
