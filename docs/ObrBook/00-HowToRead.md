导航：

- [ObrBook 目录](./README.md)

这本手册面向“严格零基础”。我会尽量满足下面这些特性：

- 每章都有“你将学会什么”（学习目标）
- 每个语法点都有“最小例子 + 逐行解释 + 常见错误”
- 示例代码会尽量满足你仓库 `demo/main.obr` 已经能跑的语法风格：`deRfun ... { ... }`、`var[type] ...;`、`std::rout(...)` 等

## 0.1 你需要准备什么

你至少需要：

- 明白 `.obr` 文件是什么：它写“函数实现”（含 `main`）
- 明白 `.mr` 文件是什么：它写“函数声明”（函数头），用于给 `.obr` 的实现做配对

如果你暂时不清楚 `.mr`：没关系，后面第 4 章会专门讲。

## 0.2 代码块约定

下面这些符号你要固定记住：

- `.obr`：源代码实现文件（扩展名为 `.obr`）
- `.mr`：模块声明文件（扩展名为 `.mr`）
- `main.obr`：入口实现文件（解释器只执行其中的 `deRfun main():void{...}`）

## 0.3 最重要的两条语法约束（先背下来）

1. 每条一般语句/表达式语句以 `;` 结尾  
   例如：`var[int] x = 0;`、`x++;`、`std::rout("hi");`

2. `if/else` 里建议你先用 `{ ... }` 块  
   例如：`if (x == 0) { std::rout("zero"); }`

后续你会理解为什么：这能避免一些“没有花括号时的终止/解析边界”带来的混淆。

## 0.4 本书的最小可运行路线

从最简单的开始：

1. 先写一个“只输出”的 `main.obr`
2. 再引入变量 `var[type] name = ...;`
3. 再引入自增 `++/--`、赋值 `=`, 以及简单表达式
4. 再引入 `if`、`while` 和三元 `?:`
5. 最后再引入 `.mr` 与模块化函数（你会知道为什么必须有 `.mr`）

准备好后，我们进入第 1 章
