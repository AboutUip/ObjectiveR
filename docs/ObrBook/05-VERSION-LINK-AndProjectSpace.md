# 05 章：`#VERSION` 与 `#LINK`（理解语言空间与可访问范围）

- [上一章：04 模块与函数](./04-ModulesMRAndFunctions.md)

这一章你将学会：

- `#VERSION` 的位置与作用
- `#LINK` 的核心语义：访问方能否访问当前文件（current file）
- `#LINK` 的两类路径项：目录空间项 与 文件路径项
- 多条 `#LINK` 的合并规则（并集）

## 5.1 `#VERSION`：告诉解释器“这份代码按哪版语法写”

你需要知道两点：

- `#VERSION` 建议写在 `.obr` 文件开头区域
- 当前语言版本在本仓库实现为 `1`

形式要求：

- 行首是 `#VERSION 1`（中间至少一个空白）
- 大小写敏感

## 5.2 `#LINK`：它控制的是“访问权限方向”

先记住一个关键措辞：

- 当前文件（`F`）里写的 `#LINK`，用来判定“访问方文件（`C`）能不能访问 `F` 的公开符号”

直觉：

- `F` 的 `#LINK` 越严格，外部就越“借不到”`F` 里的东西

## 5.3 `#LINK` 路径项只有两种类型

`#LINK` 的每个路径项只能是：

### 5.3.1 目录空间路径项（目录）

例如：

- `/`
- `/lib`
- `/lib/`

命中规则：

- 只要访问方文件 `C` 位于该目录空间及其子目录下（对应 `.obr` 的路径），就命中

### 5.3.2 文件路径项（单文件）

例如：

- `/main.obr`
- `/lib/util.obr`

命中规则：

- 访问方文件 `C` 的规范化绝对路径与该项完全相等，才命中

## 5.4 默认行为：不写 `#LINK` 等价于 `#LINK /`

- 如果你在某个 `.obr` 里根本没有写 `#LINK`，解释器会把它当作 `#LINK /`
- 也就是“项目根空间开放”，其它 `.obr` 默认都能访问它

## 5.5 多条 `#LINK`：并集（OR）

如果一个文件出现多条 `#LINK` 指令，它们的路径项合并为一个并集列表。

例如等价关系：

```text
#LINK /lib
#LINK /main.obr
```

等价于“允许访问 `/lib` 空间 + 允许访问 `/main.obr` 这个单文件”的并集效果。

## 5.6 逗号换行续写（你在大工程里会遇到）

`#LINK` 指令参数允许用“逗号后换行续写”的方式。

逗号续写条件（你只要记一句话）：

- 当前行的 `#LINK` 参数区以逗号 `,` 结束（且逗号后除了空格外没有其它字符），则下一行会作为同一 `#LINK` 参数串的续写

例子：

```text
#LINK /lib,
      /main.obr
```

## 5.7 常见用法建议（零基础阶段）

如果你只是做学习项目：

- 先把所有你会写的 `.obr` 都用 `#LINK /`
- 等你开始拆成多个“功能文件”后，再收窄 `#LINK`，用来观察访问权限带来的错误提示

这样你不会一开始就被访问控制规则“劝退”。

## 5.8 常见错误清单

- 把 `#LINK` 写到 `.mr` 里：不允许（`.mr` 不走预处理指令）
- 逗号后没有续写规则，导致下一行被当作新指令：会直接解析失败
- 把路径写成相对路径：`#LINK` 路径项是以项目根为起点的“绝对逻辑路径”，统一用 `/`
