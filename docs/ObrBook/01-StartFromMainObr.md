# 01 章：从 `main.obr` 写起（第一个可运行程序）

导航：

- [上一章：00 如何读这本书](./00-HowToRead.md)
- [下一章：02 变量、赋值与表达式](./02-VariablesAndExpressions.md)

这一章你将学会：

- 一个最小可运行的 `main.obr` 长什么样
- `#VERSION` / `#LINK` 在文件里放在哪里、它们做什么
- `deRfun main():void{ ... }` 是入口（void 函数不需要 `return` 语句）

## 1.1 你先写什么（最小例子）

把下面内容放进 `main.obr`（文件名必须严格叫 `main.obr`，大小写敏感）：

```text
#VERSION 1
#LINK /

deRfun main():void{
    std::rout("Hello Obr!");
}
```

这个程序做了两件事：

1. `main.obr` 通过预处理指令声明：语言版本与语言空间根
2. 解释器会找到 `deRfun main():void{ ... }` 作为入口，然后执行花括号里的语句

## 1.2 `#VERSION` 与 `#LINK`（先有个概念）

### `#VERSION 1`

- 用来声明这份 `.obr` 代码使用哪一版 Obr 语言语法
- 当前仓库定义并实现的版本是 `1`（所以你先写 `#VERSION 1`）
- 这一行可以不写，但为了零基础阶段更稳定，建议你写上

### `#LINK /`

`#LINK` 用来描述“语言空间访问规则”：哪些文件可以访问当前文件公开的符号。

- 先理解一句话：如果你不写 `#LINK`，默认就等价于 `#LINK /`
- `#LINK /` 的直觉：等价于“项目根空间开放”，也就是通常能让其它 `.obr` 文件访问你

这一章我们还不会深入它，但你后面第 5 章会把它讲清楚。

## 1.3 `deRfun main():void{ ... }`

`deRfun` 是函数相关的关键字。

对于入口函数：

- 函数签名必须是 `deRfun main():void`
- 返回类型是 `void`：表示“没有返回值”
- 入口执行结束的方式是：执行完 `}`（不要写 `return`）

## 1.4 注释（你写代码时一定会用到）

Obr 有两种注释：

- 单行注释：`// ...`
- 多行注释：`/* ... */`

示例：

```text
deRfun main():void{
    // 这一行不会参与语义
    std::rout("ok");
    /* 这是多行注释
       也不会参与语义 */
}
```

## 1.5 常见错误（这一章只记最常见的）

- 文件名不是 `main.obr`：解释器找不到入口
- 忘记分号 `;`：像 `std::rout("hi")` 末尾没有 `;` 就会报语法错误
- 试图给 `void` 函数写 `return ...;`：入口是 `void`，先别写 return（后面你会学到非 void 时才写返回值）
